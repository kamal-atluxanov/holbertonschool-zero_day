# Zero-Day Vulnerabilities

This repository contains notes and resources for learning about **Zero-Day vulnerabilities** and exploits in cybersecurity.


##  What is a Zero-Day?

A **Zero-Day vulnerability** is a security flaw that is **unknown to the software vendor** and has **no available patch**. 
Attackers can exploit it before the vendor has a chance to fix it, making it highly dangerous.


##  Key Concepts

### 1. **Zero-Day Exploit**
- A method or code used to take advantage of a zero-day vulnerability
- Often used in targeted attacks or malware campaigns

### 2. **Discovery & Disclosure**
- Found by security researchers or attackers 
- Responsible disclosure: Researchers notify the vendor before publicizing 
- Malicious disclosure: Attackers exploit it without informing the vendor

### 3. **Patch Management**
- Once discovered, vendors release a **security patch** to fix the vulnerability 
- Timely patching reduces risk of exploitation

### 4. **Prevention & Mitigation**
- Keep software up-to-date 
- Use intrusion detection systems (IDS) 
- Apply principle of least privilege 
- Employ network segmentation and monitoring


##  Learning Goals

- Understand what zero-day vulnerabilities are and why they are dangerous
- Learn how attackers exploit unknown flaws
- Explore responsible disclosure and patching processes
- Develop strategies to mitigate zero-day risks


**Tip:** Focus on detection, prevention, and timely patching to reduce the impact of zero-day attacks.
